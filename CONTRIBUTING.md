# Contributing to SyncSpace

First off, thank you for considering contributing to SyncSpace! It's people like you that make SyncSpace such a great tool.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issues](../../issues) if it's already there. If not, feel free to open a new one!

## Branching Strategy

We use the following branch strategy:
- `main`: Stable production code.
- `development`: Active development branch. All feature branches merge here.
- `feature/*`: Feature branches created off `development`.

## Pull Requests

1. Fork the repo and create your branch from `development`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Issue that pull request!

## Code Review

All submissions, including submissions by project members, require review. We use GitHub pull requests for this purpose. Consult [GitHub Help](https://help.github.com/articles/about-pull-requests/) for more information on using pull requests.
